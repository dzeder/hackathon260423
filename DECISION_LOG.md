Alternatives: stub servers that return empty (rejected — blocks Track A integration); wire real external APIs now (rejected — requires 5 different keys). Reason: single source of truth for runtime validation and JSON Schema discovery; satisfies engineering-standards rule "no `any` types in Track B MCP server schemas". Reason: keeps the privacy invariant testable and co-located with the tool surface, not buried in fixtures. Alternatives: enforce at fixture-load time (rejected — harder to test; harder to audit).

Alternatives: pass the URL through from the LWC (rejected — trivially bypasses the allow-list); no allow-list (rejected — any future tool added server-side would be callable without a policy review).
